Hosting python discord bot on heroku

Many discord bot creators are unable to host their bot on heroku as they get Internal Server Error when they try to integrate with github. Pc users can use Heroku CLI but phone users can't use it as it is only for windows/linux. In this article you'll get to know how you can use termux to push your changes to heroku and operate it like the Heroku CLI.

Requirements

Packages: Git, Nodejs

Others: A github repository, heroku app and obviously Termux.

Steps

1. Setting up the heroku application

2. Setting up the github repository

3. Configuring the Heroku CLI

Setting up the heroku application

1. Create a new application here or skip this step if you've done it already.

2. Go to settings and add the heroku/python buildpack.

3. Enable worker dyno.

4. In case you have to do something else too like adding config vars, do it.

Setting up the github repository

1. Create a new repository here or skip this step if you've already done it.

2. Upload your code there.

3. Create a new file with the name Procfile and add this as the content, here the name of my main file is main.py so i have it like this



4. Create a new file with the name requirements.txt, this file contains all the dependencies your bot uses. You can add all the dependencies with $ pip freeze > requirements.txt command.



4. Copy url of the repository.

5. Clone the repository locally in termux with $ git clone <url> command.

6. Cd to the repository's root folder using the $ cd foldername command.

Configuring the Heroku CLI

1. Install the npm module with $ npm i -g heroku command.

2. Execute the $ heroku login command.

3. It will open the link in a browser, login there.

4. Return to termux and create a new remote with $ heroku git:remote -a app-name command, replace app-name with the name of your heroku application.

5. Push your code on heroku with the $ git push heroku main command.
